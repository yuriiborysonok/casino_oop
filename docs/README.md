# Casino Platform Documentation

## Огляд архітектури
Цей проект реалізує бекенд міні-казино з використанням мікросервісної архітектури на мові C++. Проект створено як лабораторну роботу з ООП, що охоплює сучасні індустріальні стандарти розробки.

### Основні принципи:
1. **TDD (Test-Driven Development):** Кожен компонент спочатку покривається unit-тестами за допомогою GTest.
2. **SOLID & ООП:** Логіка розбита на слабо зв'язані класи з використанням абстракцій.
3. **Design Patterns:** Використовуються Factory, Strategy та Decorator.
4. **Мікросервіси:** Взаємодія з клієнтом відбувається через REST API.

## Структура проекту
* `src/` - вихідні коди (`.cpp` файли)
* `include/` - заголовкові файли (`.h` / `.hpp`), де оголошені інтерфейси та класи
* `tests/` - юніт-тести (позитивні та негативні сценарії)
* `docs/` - документація проекту
* `frontend/` - клієнтський веб-інтерфейс
